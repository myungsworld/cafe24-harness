세션 초반의 응답은 짧게 유지한다(5~10줄 이내). 긴 표·옵션 비교를 첫 응답에 쏟지 않는다.

**Why:** 세션 초반 긴 응답은 일부 IDE에서 렌더링이 잘려 사용자가 못 본다. 또 맥락이 덜 잡힌 상태의 장황한 제안은 헛수고가 되기 쉽다.

**How to apply:**
- 첫 응답은 핵심만 5~10줄.
- 큰 작업은 먼저 한두 줄로 방향을 제시하고, 실행 전 명시적 승인을 기다린다.
- 상세 계획이 필요하면 플랜 모드/플랜 파일을 쓰고, 본문은 요약만.
