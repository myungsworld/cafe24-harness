카페24 관리자에서 풀리는 문제(메인진열 표시 설정, 게시/미게시, 외부 위젯 등)는 **코드(CSS/JS)로 우회 구현하지 않는다.** 관리자 설정으로 안내한다.

**Why:** 예) 메인진열 슬롯의 spec 필드(상품코드/원산지/제조사/배송정보) 과다 노출은 *관리자 → 상품진열 → 메인분류 진열관리 → 표시 설정* 체크박스 영역이다. 템플릿(`{$item_display|display}`)은 관리자 설정을 그대로 렌더링한다. CSS로 강제 `display:none` 화이트리스트를 넣으면, 관리자가 향후 표시 설정을 바꿔도 반영 안 되고 다른 슬롯과 일관성이 깨진다.

**How to apply:**
- 새 메인진열 슬롯 생성 직후엔 표시 설정이 기본값(전체 표시)임을 사용자에게 먼저 안내하고, "기존 슬롯과 동일하게 표시 설정을 맞춰주세요"라고 가이드.
- 화면이 코드대로 안 나오면 `/cafe24/work admin` 또는 `/cafe24/echeck`로 어드민 상태를 먼저 의심.
- 공통 함정 상세는 cafe24-harness `knowledge/cafe24_admin.md` 참조. 관련 [[persona_board_no]].
