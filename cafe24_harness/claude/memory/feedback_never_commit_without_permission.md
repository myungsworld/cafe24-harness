description: "커밋하고" 지시가 없으면 git commit 실행 금지. 작업 완료 느낌으로도 자동 커밋 금지
metadata:
  type: feedback
---

사용자가 "커밋해" 라고 명시적으로 지시하지 않은 상태에서 `git commit`을 실행하지 않는다.

**Why:** 작업이 끝난 듯 보여도 사용자가 검토·수정하고 싶을 수 있고, 커밋 단위/메시지도 사용자 권한이다. 자동 커밋은 히스토리를 어지럽힌다.

**How to apply:**
- 작업을 마쳐도 커밋하지 말고, 변경 요약만 보고하고 멈춘다.
- 커밋이 필요해 보이면 "커밋할까요?"라고 묻는다.
- push/force-push/reset --hard 등은 더 강하게 — 명시적 지시 + 확인.
