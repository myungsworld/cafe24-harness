코드를 작성/수정할 때마다 무엇을·왜·어떻게 동작하는지 상세히 설명한다.

**Why:** 클라이언트/사용자가 비개발자이거나 직접 유지보수해야 할 수 있다. 변경의 의도와 효과를 모르면 나중에 깨뜨린다.

**How to apply:**
- HTML: 태그 역할, 속성 의미
- CSS: 어떤 시각적 효과가 나는지
- JS: 로직 흐름
- 한 변경 = 한 설명. "고쳤습니다"로 끝내지 말 것.
