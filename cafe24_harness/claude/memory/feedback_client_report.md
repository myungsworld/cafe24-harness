클라이언트에게 보내는 리포트(client-report)에는 **완료한 작업 로그를 넣지 않는다.** 클라이언트의 응답·결정·자료(이미지, 문구, 설정 권한 등)가 필요한 항목만 하나의 리포트로 합쳐서 누적한다.

**Why:** 클라이언트는 "내가 뭘 해줘야 하나"만 알면 된다. 작업 로그는 노이즈다. 어드민 설정처럼 클라이언트만 할 수 있는 항목을 명확히 모아야 진행이 막히지 않는다.

**How to apply:**
- 항목 = "확인 필요" 또는 "자료/권한 요청". 완료 보고는 PROGRESS.md에.
- Slack paste 기준 강조 표기는 `*text*`(볼드)로 통일. `_text_`(이탤릭)는 한글에서 시각 강조가 약하므로 쓰지 않는다.
- 어드민에서만 풀리는 항목(게시함 클릭, 표시설정 등)은 정확한 관리자 경로와 함께 적는다.
