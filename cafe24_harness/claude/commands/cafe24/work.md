# /cafe24/work — 카페24 개발 워크플로 (어드민 진단 포함)

PLAN → DO → CHECK 3단계로 작업하되, **카페24의 핵심 함정 — "코드를 고쳤는데 화면이 안 바뀐다 → 알고 보니 어드민 설정 문제"** 를 잡는 ADMIN 단계를 추가한 워크플로.

## 사용법

```
/cafe24/work [plan|do|check|admin|status]
```

- `plan "설명"` : 작업 계획 수립 (코드 작업 vs 어드민 설정 사전 분류)
- `do` : 현재 계획 실행
- `check` : 작업 결과 검증 (코드 + Playwright 실사이트)
- `admin` : **변경이 안 먹힐 때 — 어드민 설정 문제인지 진단** (이 하네스의 핵심)
- `status` : 현재 진행 상태
- 인자 없으면 `status`

## 대원칙 (카페24 거버넌스)

- **관리자에서 풀리는 문제를 CSS/JS로 우회하지 않는다.** 메인진열 표시설정·게시/미게시·외부 위젯(크리마) 같은 건 코드가 아니라 어드민에서 해결. 코드 우회는 향후 설정 변경과 어긋나고 다른 슬롯과 일관성도 깨진다.
- **카페24 모듈 변수(`{$...}`)는 수정 대상이 아니다** — 치환 결과만 본다.
- **읽기전용 우선**: 어드민 진단은 보고/안내만. 라이브 상태 변경(게시 토글 `setPost` 등)은 사용자 승인 후 수동.
- 공통 어드민 지식은 `cafe24_harness/knowledge/cafe24_admin.md`(= `cf` 패키지). 막히면 먼저 여기를 본다.

---

## plan — 1단계: 계획

1. 프로젝트 `CLAUDE.md`에서 작업 범위·클라이언트 요구사항·**배포 방식(paste/sftp)** 확인.
2. `PROGRESS.md`에서 현재 진행 상태 확인.
3. `cafe24-planner` 에이전트로 작업을 분해:
   - 영향 받는 파일 목록 / 작업 순서(의존성) / 예상 변경
   - **각 작업을 [코드 변경] vs [어드민 설정] 으로 사전 분류.** 어드민 설정으로 풀리는 건 코드 작업에서 빼고 "어드민 안내"로 표시.
4. 계획을 사용자에게 제시하고 확인.
5. 확인 후 `PROGRESS.md`에 작업 항목 추가.

## do — 2단계: 실행

1. 계획에 따라 코드 작성/수정.
2. **각 변경마다 상세 설명** (피드백 규칙 준수):
   - HTML: 태그 역할·속성 의미 / CSS: 시각적 효과 / JS: 로직
3. 이미지 `src`는 카페24 서빙 경로(`/web/upload/<폴더>/`)로. 로컬 레포 경로(`weskin_import/img/...`) 금지.
4. 파일 변경 후 `PROGRESS.md` 작업 로그에 기록.
5. **커밋은 사용자가 명시적으로 지시할 때만.** 작업이 끝난 느낌으로 자동 커밋 금지.

## check — 3단계: 검증

1. `cafe24-evaluator` 에이전트로 검수 (HTML/CSS/JS/이미지/요구사항 충족).
2. Playwright 실사이트 검증 — 변경이 실제로 반영됐는지 (필요하면 `/cafe24/echeck`).
3. **변경이 반영 안 됐으면 곧장 `admin` 단계로** (코드는 맞는데 화면이 안 바뀌는 전형적 케이스).
4. 통과 시 `PROGRESS.md` 항목 체크.

## admin — ★ 어드민 설정 진단 (이 하네스의 핵심)

"코드 고쳤는데 화면이 안 바뀜" / "이거 코드로 되나?" 싶을 때 실행. **목표: 원인을 [코드 / 어드민 설정 / 외부 위젯] 으로 판정**.

1. **공통 함정 대조** — `knowledge/cafe24_admin.md`의 알려진 패턴과 현재 증상을 맞춰본다:
   - 메인/게시판 후기가 안 뜸 → **네이버페이 비회원 후기는 기본 "미게시"** (게시함 버튼 눌러야 노출).
   - 메인 리뷰가 게시/고정/캐시삭제로도 안 바뀜 → **외부 위젯(크리마 cre.ma)이 카페24 게시판을 시각적으로 덮음** → 크리마 어드민에서 설정.
   - 메인진열 슬롯에 상품코드/원산지 등 spec 필드 과다 노출 → **어드민 상품진열 표시설정** (CSS로 숨기지 말 것).
2. **어드민 실제 상태 확인** (세션 있을 때 — `cf doctor`로 확인):
   - 후기/게시판 문제: `cf inspect board` → 미게시 글번호 출력.
   - 설정 페이지 확인: `cf open boards` (게시판 목록) / `cf open <url|/path|alias>` 로 해당 설정 화면 DOM·스샷 덤프.
   - 세션 없으면: `cf login` 안내(사람이 headed 브라우저로 직접 로그인).
3. **외부 위젯 확인**: 메인 리뷰가 안 바뀌면 페이지에 `widgets.cre.ma` / `review*.cre.ma` 스크립트가 떠 있는지 확인 (`/cafe24/echeck` storefront 측면).
4. **판정 & 안내**:
   - 코드 문제 → `do`로 돌아가 수정.
   - **어드민 설정 문제 → 정확한 설정 경로를 사용자에게 안내** (예: "관리자 → 상품진열 → 메인분류 진열관리 → 표시 설정에서 체크박스 조절"). 코드 우회 금지.
   - 외부 위젯 → 해당 솔루션(크리마 등) 어드민에서 처리하라고 안내.
5. 새로 알게 된 **모든 몰 공통** 지식이면 `knowledge/cafe24_admin.md`에 누적(→ push 시 전 프로젝트 전파). **이 몰 고유**값(board_no 등)이면 페르소나(`admin/config.yaml`)/프로젝트 메모리에 기록.

## status — 현재 상태

1. `PROGRESS.md` 읽기.
2. 마일스톤별 완료율 표시.
3. 다음 할 일 제안 (어드민 확인 대기 항목이 있으면 함께 표시).
